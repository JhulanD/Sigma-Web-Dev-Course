Tutorial #21-CSS Specificity & Cascade

Here is the standard ranking of **CSS selector specificity**, from lowest to highest: 

1. **Universal selector** (`*`)
2. **Element (type) selectors** (e.g., `h1`, `p`) and **pseudo-elements** (`::before`, `::after`)
3. **Class selectors** (e.g., `.classname`), **attribute selectors** (e.g., `[type="text"]`), and **pseudo-classes** (e.g., `:hover`)
4. **ID selectors** (e.g., `#id`)
5. **Inline styles** (e.g., `style="..."` inside an element attribute; highest specificity)

**Note:**  
- Class, attribute, and pseudo-class selectors all have **equal specificity** and are ranked above element selectors but below ID selectors.
- If multiple selectors have the same specificity, the one defined last in the CSS applies.
- The `!important` rule can override specificity, but only within the same origin/cascade layer.[1][2][3][4][5]

**Specificity points (for calculation):**
- Inline style: 1000
- ID selector: 100
- Class/attribute/pseudo-class: 10
- Element/pseudo-element: 1
- Universal: 0

So, in order:  
**Inline style > ID > Class/attribute/pseudo-class > Element/pseudo-element > Universal**.[2][3][4][5][1]
